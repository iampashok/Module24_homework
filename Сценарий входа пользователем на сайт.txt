# Сценарий входа пользователем на сайт


**Цель:** Пользователю нужно зайти на главную страницу сайта, чтобы сделать заказ
**Акторы:** Пользователь, система (сайт)

__*Основной сценарий:*__

1. Пользователь заходит на сайт через электронный адрес:

[TP-1: Заход на сайт пиццерии через электронный адрес](https://iampashok.atlassian.net/browse/TP-1)



__*Альтернативный сценарий 1:*__

1.1. Пользователь заходит на сайт найдя его по названию через поисковик:

[TP-2: Заход на сайт пицерии найдя её по названию через поисковик](https://iampashok.atlassian.net/browse/TP-2)



 
__*Основной сценарий:*__

2. При заходе пользователем на сайт система определяет город из которого заходит пользователь 
[TP-5: Выбор города при заходе на сайт](https://iampashok.atlassian.net/browse/TP-5)



__*Альтернативный сценарий:*__


2.1. Системе не удалось определить город пользователя, по-умолчанию городом пользователя назначается Москва



3. Сайт выводит окно “Ваш город … ?” с вариантами ответа “Да и “Нет”

[TP-6: Вывод окна "Ваш город ...?" при заходе на сайт](https://iampashok.atlassian.net/browse/TP-6)

__*Основной сценарий:*__


4. Пользователь выбирает ответ “Да”, то система скрывает окно “Ваш город…?”.



__*Альтернативный сценарий:*__


4.1. Пользователь выбирает ответ “Нет”, 


4.2. Система показывает список доступных городов для выбора пользователем.


4.3. Пользователь выбирает свой город из списка



5. После выбора подтверждения пользователем города становиться доступно главное окно сайта 
[TP-26: Выбор пиццы и топпингов на главной странице](https://iampashok.atlassian.net/browse/TP-26)

**Исключения:** Города пользователя нет в списке п. 4.2., в этом случае пользователь понимет, что доставки в его город нет, тогда он выбирает любой город чтобы попасть на сайт и ознакомиться с ассортиментом, либо уходит с сайта
